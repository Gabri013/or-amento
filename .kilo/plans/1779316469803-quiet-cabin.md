# Plano de Migração — Cozinca Inox (PHP → React + Firebase)

## 1. Visão Geral

| Item | Origem | Destino |
|---|---|---|
| Framework | PHP 8 + MySQL + XAMPP | React 18 + TypeScript + Vite |
| Estilização | Bootstrap 5 + CSS inline | Tailwind CSS 3 |
| Banco de dados | MySQL (relacional) | Firebase Firestore (NoSQL) |
| Autenticação | Session PHP (`$_SESSION`) | Firebase Auth (email + senha) |
| Rotas | Links `<a href>` por arquivo | React Router v6 |
| Gráficos | Chart.js | Recharts |
| PDF | HTML + print CSS | @react-pdf/renderer |
| Upload de imagens | `move_uploaded_file` para `/uploads/` | Firebase Storage |
| Deploy | XAMPP local | Firebase Hosting |

## 2. Mapeamento do Banco (MySQL → Firestore)

### Tabelas → Coleções Firestore

| MySQL | Firestore | Notas |
|---|---|---|
| `usuarios` | `usuarios` | `id` vira `uid` (Firebase Auth UID) |
| `orcamentos` | `orcamentos` | `id` vira `id` autogerado |
| `orcamento_itens` | **merge em `orcamentos.itens[]`** | Subcoleção vira array embutido |
| `clientes` | `clientes` | `id` autogerado |
| `produtos` | `produtos` | `id` autogerado; imagem URL em vez de path |
| `vendas` | `vendas` | `id` autogerado |
| `venda_itens` | **merge em `vendas.itens[]`** | Subcoleção vira array embutido |

**Decisão:** Os itens de orçamento e venda são armazenados como **array embutido** dentro do documento pai, não como subcoleção separada. Isso simplifica leituras e evita transações complexas.

## 5. Lógica que mudou (PHP → Firestore)

### Código do Orçamento

**PHP:** `SELECT COUNT(*) + 1 WHERE id_usuario = ?`
```sql
SELECT COUNT(*) as total FROM orcamentos WHERE id_usuario = ?
```

**Firestore:**
```ts
const q = query(collection(db, COLECOES.ORCAMENTOS), where('vendedorId', '==', uid));
const snapshot = await getDocs(q);
const numero = str_pad(snapshot.size + 1, 3, '0');
const codigo = numero + inicialDoNome;
```

### Filtro por mês (Dashboard)

**PHP:** `WHERE MONTH(data_criacao) = MONTH(CURRENT_DATE())`
```ts
const agora = Timestamp.now();
const inicioMes = new Date(agora.toDate().getFullYear(), agora.toDate().getMonth(), 1);
const fimMes = new Date(agora.toDate().getFullYear(), agora.toDate().getMonth() + 1, 0, 23, 59, 59);
const q = query(
  collection(db, COLECOES.ORCAMENTOS),
  where('criadoEm', '>=', Timestamp.fromDate(inicioMes)),
  where('criadoEm', '<=', Timestamp.fromDate(fimMes))
);
```

### Cálculo de totais

**PHP (criar_orcamento.php JS):**
```js
totalProdutos = Σ (qtd × precoUnitario)
totalGeral     = totalProdutos + frete
valorDesconto  = totalGeral × (desconto / 100)
totalFinal     = totalGeral - valorDesconto
```
**React:** mesma lógica, pura — sem servidor envolvido.

### Upload de imagem de produto (Firebase Storage)

```ts
const storageRef = ref(storage, `produtos/${produtoId}/imagem_${fileName}`);
const snapshot = await uploadBytes(storageRef, file);
const url = await getDownloadURL(snapshot.ref);
// salvar `url` no campo `imagem` do documento Firestore
```

## 7. Regras do Firestore (firestore.rules)

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    function isLoggedIn() {
      return request.auth != null;
    }

    function isAdmin() {
      return isLoggedIn() &&
        get(/databases/$(database)/documents/usuarios/$(request.auth.uid)).data.tipo == 'admin';
    }

    function isOwner(vendedorId) {
      return isLoggedIn() && request.auth.uid == vendedorId;
    }

    match /usuarios/{uid} {
      allow read: if isLoggedIn() && (isAdmin() || request.auth.uid == uid);
      allow write: if isAdmin();
    }

    match /orcamentos/{id} {
      allow read: if isAdmin() || isOwner(resource.data.vendedorId);
      allow create: if isLoggedIn();
      allow update, delete: if isAdmin() || isOwner(resource.data.vendedorId);
    }

    match /vendas/{id} {
      allow read: if isAdmin() || isOwner(resource.data.vendedorId);
      allow create: if isLoggedIn();
      allow update, delete: if isAdmin() || isOwner(resource.data.vendedorId);
    }

    match /clientes/{id} {
      allow read, write: if isLoggedIn();
    }

    match /produtos/{id} {
      allow read, write: if isLoggedIn();
    }
  }
}
```

## 8. Cloud Function — criarUsuario

```ts
// functions/src/index.ts
import * as functions from 'firebase-functions';
import * as admin from 'firebase-admin';
admin.initializeApp();

export const criarUsuario = functions.https.onCall(async (data, context) => {
  const uid = context.auth?.uid;
  if (!uid) throw new functions.https.HttpsError('unauthenticated', 'Não autenticado');
  const callerDoc = await admin.firestore().collection('usuarios').doc(uid).get();
  if (callerDoc.data()?.tipo !== 'admin') {
    throw new functions.https.HttpsError('permission-denied', 'Apenas admins');
  }
  const { email, senha, nome, tipo } = data;
  const userRecord = await admin.auth().createUser({ email, password: senha });
  await admin.firestore().collection('usuarios').doc(userRecord.uid).set({
    uid: userRecord.uid,
    nome,
    email,
    tipo,
    ativo: true,
    criadoEm: admin.firestore.FieldValue.serverTimestamp(),
  });
  return { uid: userRecord.uid };
});
```

## 9. Script de Seed

```ts
// src/firebase/seed.ts
import { getFirestore, collection, getDocs, addDoc, serverTimestamp, query, limit } from 'firebase/firestore';
import { app } from './config';

const db = getFirestore(app);

const PRODUTOS_INICIAIS = [
  { nome: 'Bancada Inox 1,00m', sku: 'BNI-100', precoBase: 850, unidade: 'un', ativo: true },
  { nome: 'Pia Inox Dupla', sku: 'PID-001', precoBase: 1200, unidade: 'un', ativo: true },
  { nome: 'Prateleira Inox', sku: 'PRI-050', precoBase: 320, unidade: 'un', ativo: true },
];

const CLIENTE_INICIAL = {
  nome: 'Restaurante Exemplo LTDA',
  documento: '00.000.000/0001-00',
  telefone: '(31) 99999-9999',
  email: 'contato@exemplo.com.br',
  endereco: 'Rua Exemplo, 100, BH/MG',
};

export async function seedBancoDeDados(): Promise<void> {
  if (localStorage.getItem('cozinca_seed_done') === 'true') return;

  const usuariosSnap = await getDocs(query(collection(db, 'usuarios'), limit(1)));
  if (!usuariosSnap.empty) {
    localStorage.setItem('cozinca_seed_done', 'true');
    return;
  }

  for (const produto of PRODUTOS_INICIAIS) {
    await addDoc(collection(db, 'produtos'), { ...produto, criadoEm: serverTimestamp() });
  }
  await addDoc(collection(db, 'clientes'), { ...CLIENTE_INICIAL, criadoEm: serverTimestamp() });

  localStorage.setItem('cozinca_seed_done', 'true');
}
```

## 10. Checklist de Pré-requisitos Firebase

Antes de qualquer código funcionar, o usuário precisa:
1. Criar projeto no Firebase Console
2. Ativar **Authentication** (Email/Password)
3. Ativar **Firestore Database** (modo produção)
4. Ativar **Storage**
5. Copiar chaves para `.env`
6. Registrar um usuário admin manualmente no Firebase Console → Auth
7. Fazer `npm run dev`

## 12. Observações de Implementação

### Comportamento do dashboard.php
- Exibe `$_SESSION['usuario']` (username string, ex: "camille", "nilton")
- Dashboard agrega dados de todos os vendedores se admin, só do próprio se usuário
- Gráfico de **linha** (`#ff530d` laranja) → valor por mês
- Gráfico de **barra** (`#1c1c1c`) → quantidade por mês
- Duas canvases `<canvas id="graficoValor">` e `<canvas id="graficoQtd">`

### Dashboard aguardar welcome.php (intervalo de 1 semana semanal)
- Não implementar no MVP

### Sobre as vendas
- `vendas.php original` → `/vendas` no React
- `vendas_detalhe.php` e `imprimir_venda.php` → mergem em modal/detalhe na página Vendas
- O PDF de venda reutiliza o mesmo template do orçamento — não implementar template separado no MVP

### Cadastro.js
- O `cadastro.php` original tem sistema de AJAX inline para edição de clientes
- No React, a edição inline vira state local + `atualizarDocumento()` — sem AJAX
